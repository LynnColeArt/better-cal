## Contract Source Types

The current codebase does not have one clean contract layer. Contracts are spread across:

- **tRPC routers and Zod schemas** for web UI calls: `../../../reference/packages/trpc/server/routers`.
- **Nest controllers and class DTOs** for API v2: `../../../reference/apps/api/v2/src` and `../../../reference/packages/platform/types`.
- **Next.js app API routes** for legacy and product-specific endpoints: `../../../reference/apps/web/app/api`.
- **Prisma schema and zod-utils** for persistence contracts and JSON sub-shapes: `../../../reference/packages/prisma/schema.prisma`.
- **Feature services** for domain side effects: `../../../reference/packages/features`.
- **App-store schemas and metadata** for integration-specific contracts: `../../../reference/packages/app-store`.
- **Webhook payload builders** for external event contracts: `../../../reference/packages/features/webhooks`.

## Areas Requiring Golden Tests

Before replacing implementation, capture black-box fixtures for:

- login/session read, profile switching, and auth failures;
- event type create/update/read for personal, team, managed, and round-robin events;
- slot lookup across timezones, buffers, selected calendars, OOO, holidays, and seated events;
- booking create/cancel/reschedule/confirm for free, paid, recurring, seated, team, and platform bookings;
- calendar/conferencing connection and booking side effects using mocked providers;
- webhook delivery payloads and signatures for every trigger in `WebhookTriggerEvents`;
- API v2 auth using API key, platform OAuth client headers, platform access token, third-party access token, and NextAuth session;
- cron/job endpoints and idempotency.
