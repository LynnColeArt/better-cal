# Registries

Registries are machine-readable inputs for contract capture and implementation.

Initial files:

- `routes.json`: public route and procedure registry.
- `policies.json`: route policy and authorization registry.
- `identifiers.json`: public identifier registry.
- `enums.json`: enum string registry.
- `secrets.json`: secret classification registry.
- `json-shapes.json`: stored JSON shape registry.

Registries start in `draft`. They should become `accepted` domain by domain after fixture capture and review.
